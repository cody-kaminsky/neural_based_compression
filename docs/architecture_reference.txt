Target: 1080p @ 30fps, 200 MHz clock, ZU3EG or equivalent

================================================================================
1. OVERVIEW
================================================================================

This FPGA encoder implements a learned image compression pipeline based on a
convolutional autoencoder with a scale hyperprior entropy model (Ballé et al.,
NeurIPS 2018). It is designed as a drop-in replacement for the DCT-based encoder
in ../architecture_reference.txt, sharing the same AXI-Stream video input and
bitstream output interfaces.

Key design choices vs. naive neural compression:

  Strip-based processing
    Input is processed in 64-row horizontal strips rather than full frames.
    Eliminates the full-frame startup latency caused by the conv receptive field.
    Strip height (64 rows) is chosen to exceed the vertical receptive field of
    the 3-stage analysis network (~30 pixels) with margin for border padding.

  Factorized + scale hyperprior entropy model, no autoregressive context
    Autoregressive (pixel-sequential) context models are fundamentally serial
    and cannot be parallelised on FPGA. The scale hyperprior provides most of
    the coding gain of the autoregressive model at much lower cost.
    Each latent channel's distribution is parameterised by a per-symbol scale
    factor derived from the hyperprior — this can be computed strip-by-strip
    and looked up per symbol, enabling fully streaming entropy coding.

  Depthwise separable convolutions (DSConv)
    Replaces all 3-stage standard conv layers with depthwise separable blocks
    (depthwise 3x3 + pointwise 1x1), reducing MAC count ~9x versus standard
    convolution at minimal quality cost.

  INT8 quantization (weights and activations)
    Post-training INT8 quantization for the conv backbone, validated via
    quantization-aware training (QAT) using the CompressAI framework.
    Two INT8 multiplies per DSP48E2 using cascade packing.

  Parallel ANS entropy coding (64 streams)
    Asymmetric Numeral Systems (ANS) replaces arithmetic coding.
    64 independent ANS encoder instances operate on interleaved latent
    channels in parallel, reducing entropy coding latency ~64x.

  Input:   AXI-Stream 8-bit YUV 4:2:0, up to 1080p (3840x2160 future)
  Output:  AXI-Stream compressed bitstream (bytes, frame-delimited)
  Control: AXI-Lite register interface (from Zynq PS)

  No motion estimation. Intra-only (I-frame only) in v1.0.
  P-frame temporal coding is planned for v2.0 (lightweight optical flow).

  Python / C++ using CompressAI or custom ONNX runtime
  ANS decode → entropy decode → synthesis_net → reconstructed pixels

3. MODULE DESCRIPTIONS
================================================================================

--------------------------------------------------------------------------------
strip_ctrl.vhd — Strip Sequencer (~150 lines)
--------------------------------------------------------------------------------
Coordinates the 64-row strip pipeline. Monitors AXI-Stream video input and
asserts strip_rdy when pixel_buffer has accumulated 64 complete rows.
Drives handshaking between pixel_buffer, analysis_net, and ans_encoder.

Strip overlap: last 2 rows of strip N are prepended to strip N+1 as border
context for the stage-0 convolution (avoids discontinuity at strip boundaries).
This requires pixel_buffer to retain the last 2 rows after each strip flush.

Signals:
  strip_start    Pulse: begin processing current strip
  strip_done     Pulse from ans_encoder: strip bitstream complete
  strip_y        Current strip's top Y coordinate (for metadata header)
  last_strip     Asserted on final strip of frame

FSM: IDLE → ACCUMULATE → WAIT_HYPER → ENCODE_Y → FLUSH

Resource estimate: ~200 LUTs, ~150 FFs, 0 DSP, 0 BRAM

--------------------------------------------------------------------------------
pixel_buffer.vhd — Input Strip Buffer (~180 lines)
--------------------------------------------------------------------------------
Dual-port BRAM buffer. Write side accepts AXI-Stream pixel input (one byte/clock,
YUV 4:2:0 raster order). Read side provides 8-pixel-wide rows to analysis_net
stage 0 on demand.

Stores 64 rows × frame_width pixels (Y channel) plus 32 rows × frame_width/2
(Cb and Cr at 4:2:0). Currently processes luma only; chroma path planned v1.1.

Capacity: 64 × 1280 × 1B (luma) = 80 KB → 2× RAMB36E2 (ping-pong)
The two RAMB36E2 alternate: one fills while the other drains to analysis_net.

Resource estimate: ~150 LUTs, ~100 FFs, 0 DSP, 4 BRAM36

--------------------------------------------------------------------------------
analysis_net.vhd — 3-Stage Analysis Transform (~600 lines)
--------------------------------------------------------------------------------
Streaming convolutional encoder. Processes pixel_buffer output one row at a time
using line-buffer-based convolution (no full feature map buffering required).

Architecture per stage:
  Line buffer (2 rows × width × channels × INT8) stored in BRAM
  Depthwise conv (3x3, one filter per input channel) using DSP MAC array
  Leaky ReLU activation (slope 0.1 for x<0) — trivial in LUTs
  Pointwise conv (1x1, Cin × Cout) using DSP MAC array
  Stride-2 subsampling: emit every other output row/column

Stage dimensions and throughput (at 200 MHz, 1080p):
  Stage 0: in  1280×64×3    out  640×32×64    ~20 DSPs
  Stage 1: in   640×32×64   out  320×16×96    ~ 6 DSPs
  Stage 2: in   320×16×96   out  160× 8×128   ~ 3 DSPs

Line buffers (BRAM):
  Stage 0 input  → stage 1:  3 rows × 640 × 64ch × 1B = 123 KB → 27 BRAM36
  Stage 1 output → stage 2:  3 rows × 320 × 96ch × 1B =  92 KB → 20 BRAM36
  Stage 2 output (latent y): 1 strip ×160 × 128ch × 1B =  20 KB →  5 BRAM36

Weight storage: INT8 weights loaded from weight_rom at startup into local BRAM
  Stage 0 depthwise:   3×3×3    =      27 B
  Stage 0 pointwise:   3×64     =     192 B
  Stage 1 depthwise:   3×3×64   =     576 B
  Stage 1 pointwise:   64×96    =   6,144 B
  Stage 2 depthwise:   3×3×96   =     864 B
  Stage 2 pointwise:   96×128   =  12,288 B
  Total analysis:               ~  20 KB → 5 BRAM36

Resource estimate: ~12,000 LUTs, ~4,000 FFs, ~30 DSP, ~57 BRAM36

--------------------------------------------------------------------------------
hyper_analysis.vhd — Hyperprior Analysis (~200 lines)
--------------------------------------------------------------------------------
2-stage sub-network that encodes the latent y into a compact side-information
vector z. Runs once per strip immediately after analysis_net completes.

Architecture: abs(y) → Conv 3x3/1 (128→64) → ReLU → Conv 3x3/2 (64→64) → z

z dimensions per strip: 160×8×64 → 80×4×64 (after stride-2 stage)
z is quantized and entropy-coded using the factorized entropy model (separate
simpler lookup-table based coder, not the ANS array used for y).
z bitstream is prepended to the y bitstream in each strip packet so the decoder
can reconstruct the scale parameters before decoding y.

Weight storage: ~8 KB → 2 BRAM36
Line buffer:    3 rows × 80 × 64ch × 1B = 15 KB → 4 BRAM36

Resource estimate: ~2,000 LUTs, ~800 FFs, ~8 DSP, ~6 BRAM36

--------------------------------------------------------------------------------
hyper_synthesis.vhd — Hyperprior Synthesis (~200 lines)
--------------------------------------------------------------------------------
Decodes z back into per-symbol scale parameters σ for the entropy model.
Mirrors hyper_analysis: upsamples z through 2 transposed conv stages.
Output: 160×8×128 scale map σ (one value per latent symbol ŷ).

σ values are quantized to 8 bits (log-scale) for compact storage during
the y encoding pass. Stored in a 160×8×128 BRAM as the y encoding proceeds.

Resource estimate: ~2,000 LUTs, ~800 FFs, ~8 DSP, ~8 BRAM36

--------------------------------------------------------------------------------
quantizer.vhd — Uniform Quantizer (~80 lines)
--------------------------------------------------------------------------------
Rounds real-valued latent activations to nearest integer: ŷ = round(y).
Implemented as a simple round-half-up on the fractional fixed-point
representation from analysis_net (Q8.8 internal representation).
No division required. 1-cycle latency per symbol.

Also computes the residual r = y - ŷ for potential soft-quantization paths
(planned future work).

Resource estimate: ~100 LUTs, ~50 FFs, 0 DSP, 0 BRAM

--------------------------------------------------------------------------------
entropy_params.vhd — Per-Symbol Parameter Lookup (~120 lines)
--------------------------------------------------------------------------------
For each quantized symbol ŷ[c,x,y], looks up the corresponding scale parameter
σ[c,x,y] from hyper_synthesis output. Computes the probability mass:

  P(ŷ) = CDF(ŷ + 0.5 | 0, σ) - CDF(ŷ - 0.5 | 0, σ)

where CDF is the Gaussian (or Laplace) CDF, pre-tabulated for 256 σ values
and 512 symbol values in a 256×512 lookup ROM (131 KB — split across BRAM).

Output: 12-bit probability p[c,x,y] in Q0.12 fixed point for ANS encoder.

Gaussian CDF ROM: 256 × 512 × 12-bit = 196,608 bytes → ~48 BRAM18 = 24 BRAM36

Note: This is the largest single BRAM consumer. Can be reduced to ~12 BRAM36
by using Laplace instead of Gaussian (symmetric, needs only half the table)
or by interpolating a coarser table (64 σ entries, interpolate at runtime).

Resource estimate: ~500 LUTs, ~200 FFs, 0 DSP, ~24 BRAM36

--------------------------------------------------------------------------------
ans_encoder.vhd — 64-Stream Parallel ANS Entropy Coder (~400 lines)
--------------------------------------------------------------------------------
Core entropy coding engine. Implements 64 independent rANS (range ANS) encoder
instances operating on interleaved latent channels.

The 128 latent channels are round-robin assigned to 64 ANS streams (2 channels
per stream). Each stream encodes symbols independently; outputs are interleaved
in the bitstream so the decoder can reconstruct them with 64 parallel decoders.

Per-stream rANS state: 32-bit integer x in [L, bL) where L = 2^23, b = 2^8
Encoding step: x' = (x / p) * M + offset(sym, p)
  where p = probability from entropy_params, M = 2^12 (probability precision)

Each ANS encoder instance: ~100 LUTs + 1 DSP (for the division-by-p step,
implemented as multiply-by-reciprocal using a 256-entry LUT).

Output: 64 streams of variable-length bitstream chunks, multiplexed to
bs_packer in round-robin order.

Latency: One symbol per clock per stream = 1 clock throughput at full pipeline.
Total entropy coding latency for one strip (160×8×128 symbols = 163,840):
  163,840 / 64 streams = 2,560 clocks ≈ 12.8 µs at 200 MHz.

Resource estimate: ~8,000 LUTs, ~3,000 FFs, ~64 DSP, ~8 BRAM36

--------------------------------------------------------------------------------
weight_rom.vhd — Packed INT8 Weight ROM (~100 lines)
--------------------------------------------------------------------------------
Stores all network weights for analysis_net, hyper_analysis, hyper_synthesis in
a single packed BRAM array. Weights are loaded into local BRAMs of each
sub-module at startup (one-time configuration) via a DMA-style sequencer.

Weight packing: 4 INT8 weights per 32-bit BRAM word.
Total weights (analysis + hyper): ~48 KB → 12 BRAM36.

At startup the PS ARM core can update weights via AXI-Lite DMA to support
switching between multiple trained models (different rate-distortion points
or scene-adapted models).

Resource estimate: ~200 LUTs, ~100 FFs, 0 DSP, 12 BRAM36

4. DATA FLOW
================================================================================

  Video Input (AXI-Stream 8-bit YUV 4:2:0)
        |
        v
  pixel_buffer  [Accumulate 64 rows; ping-pong BRAM]
        |
        | strip_start pulse (every 64 rows)
        v
  analysis_net  [3-stage DSConv, streaming row-by-row]
        |
        +--[y latent, 160×8×128 INT8]----> quantizer -----> ŷ stored in BRAM
        |                                                         |
        v                                                         |
  hyper_analysis  [2-stage, runs on ŷ]                           |
        |                                                         |
        v                                                         |
  [z latent, 80×4×64 INT8]                                        |
        |                                                         |
        v                                                         |
  [factorized entropy code z → z bitstream]                       |
        |                                                         |
        v                                                         |
  hyper_synthesis  [decode z → scale map σ, 160×8×128]            |
        |                                                         |
        v                                                         v
  entropy_params  [compute P(ŷ) from σ] <-------- ŷ symbols ------+
        |
        v
  ans_encoder  [64 parallel rANS streams encode ŷ symbols]
        |
        v
  bs_packer  [frame header + strip packets → AXI-Stream bitstream output]


  Timing per strip (64 rows of 1280px input):
    pixel_buffer fill:      64 × 1280 / 1px_per_clk = 81,920 clocks (partially
                            hidden behind previous strip's ANS encode)
    analysis_net:           ~10,000 clocks (streaming, dominated by stage-0)
    hyper_analysis:         ~2,000 clocks
    hyper_synthesis:        ~2,000 clocks
    entropy_params lookup:  ~163,840 / pipelined ≈ 2,000 clocks
    ans_encoder:            ~2,560 clocks
    -------------------------------------------------------
    Total exposed latency:  ~20,000 clocks ≈ 0.1 ms per strip at 200 MHz

  Full frame (17 strips of 64 rows at 1080p):
    17 × 0.1 ms ≈ 1.7 ms encode latency (pipelined; strips overlap with fill)

switching to Laplace CDF and exploiting symmetry, reducing total BRAM to ~107
(49% utilisation). Recommended if the v2.0 P-frame extension is planned.

7. BITSTREAM FORMAT
================================================================================

Frame header (per frame):
  [8 bits]  magic         0xNE (Neural Encoder)
  [16 bits] frame_width
  [16 bits] frame_height
  [8 bits]  model_id      identifies rate-distortion operating point
  [8 bits]  num_strips    ceil(frame_height / 64)

Per-strip packet:
  [16 bits] strip_y       top row of this strip
  [16 bits] z_len         byte length of z (hyperprior) bitstream
  [16 bits] y_len         byte length of y (main latent) bitstream
  [z_len bytes]  z_bitstream   ANS-coded hyperprior side information
  [y_len bytes]  y_bitstream   64-stream interleaved ANS-coded latent

The z bitstream uses a simpler factorized entropy code (not ANS) since z is
small (~80×4×64 symbols per strip). This simplifies the decoder.
The y bitstream is 64-stream interleaved rANS; decoder requires 64 parallel
rANS decoders (trivially parallelisable on CPU/GPU ground station).

================================================================================
10. TRAINING AND WEIGHT EXPORT WORKFLOW
================================================================================

1. Train analysis_net + hyper_analysis + hyper_synthesis using CompressAI
   framework on aerial video dataset (UAVid, VisDrone, or proprietary).
   Loss: L = R + λD  where R = estimated bitrate, D = MS-SSIM or MSE.
   Train 3 models at λ = {0.01, 0.05, 0.10} for 3 quality operating points.

2. Quantization-aware training (QAT): fine-tune with simulated INT8
   quantization for 10–20 epochs. Target: < 0.3 dB PSNR degradation vs FP32.

3. Export weights: run export_weights.py to produce .mem files (one per layer)
   in the format expected by weight_rom.vhd initialization.

4. Pack weights into weight_rom: model_id 0/1/2 occupy separate address ranges
   in weight_rom. The PS ARM can switch models at runtime by writing MODEL_ID
   and pulsing WEIGHT_DMA to reload the MAC arrays.

================================================================================
11. V2.0 PLANNED EXTENSIONS
================================================================================

Temporal P-frame coding (single reference, low-delay):
  - Add lightweight optical flow module (2-level SpyNet, depthwise separable)
  - Warp previous reconstructed strip from DDR reference buffer
  - Encode residual through analysis_net (same weights, different λ)
  - Requires: ~20 BRAM36 (flow feature buffers), ~20 DSP (flow network),
              ~3,000 LUTs → still fits ZU3EG with headroom

Chroma encoding:
  - Currently luma-only. Add parallel 4:2:0 chroma path.
  - Chroma at half resolution; use same analysis_net weights (finetuned)
  - Requires: ~15 BRAM36 additional line buffers

Resolution scaling to 4K:
  - Line buffer BRAM scales linearly with frame_width. At 3840px:
    Stage 0 line buffer triples (640→1920px). Requires ZU5EV or larger device.
